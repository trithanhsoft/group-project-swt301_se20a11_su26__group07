RECIPE SEED GUIDE - MINI COFFEE POS SWT
========================================

Muc dich file nay
-----------------
File nay ghi chu cach import du lieu cong thuc do an/uong tu du an cu sang database moi cua Mini Coffee POS SWT.
Dung de bo vao thu muc document/docs trong project cho team hieu ro logic import cong thuc.

Pham vi hien tai cua database
-----------------------------
Project hien tai chi giu dung scope SWT:
- Login / Role Admin va Staff
- Product Management
- Ingredient Management
- Stock Import / Stock Adjust
- Recipe Management
- POS Order
- Order History
- Reports

Database moi khong dung cac bang/cu phap cua project cu nhu:
- product_variants
- product_recipes
- stores
- shifts
- customers
- roles phuc tap
- inventory_transactions theo tung cua hang
- sp_add_recipe
- parent_id / processing_ratio / backflush

Cac bang cong thuc trong database moi
-------------------------------------
Cong thuc hien duoc map vao cac bang sau:

1. products
   Luu mon ban. Vi database moi khong co bang product_variants nen moi size se thanh mot product rieng.
   Vi du:
   - Phin Den Da - S
   - Phin Den Da - M
   - Phin Den Da - L

2. ingredients
   Luu nguyen lieu de tru kho.
   Cac ingredient_code trong file cu se duoc map thanh ingredient name trong database moi.
   Vi du:
   - PREP_CF_PHIN -> Cot cafe phin
   - PREP_SUG -> Nuoc duong
   - RAW_CF_ESP -> Cafe hat Espresso
   - LY_S -> Ly nhua S

3. recipes
   Moi product co mot recipe active.

4. recipe_items
   Luu tung dong nguyen lieu trong cong thuc.
   Moi dong gom:
   - recipe_id
   - ingredient_id
   - quantity_required

Nguyen tac map size
-------------------
Do database moi khong co product_variants, size S/M/L se duoc gan vao ten product.

Cong thuc cu:
- product_name: Phin Den Da
- size: S

Sang product moi:
- name: Phin Den Da - S

Nguyen tac map don vi
---------------------
Database moi dang dung enum ingredient_unit gom:
- GRAM
- ML
- PIECE

Map tu file cu:
- gr   -> GRAM
- ml   -> ML
- cai  -> PIECE
- trai -> PIECE
- cay  -> PIECE

Luu y ve PREP / JELLY
---------------------
Trong du an cu, mot so nguyen lieu co logic so che/backflush:
- PREP_CF_PHIN
- PREP_SUG
- JELLY_CF
- JELLY_GRN
- JELLY_PCH
- JELLY_LYC

Database moi khong dung parent_id, processing_ratio, is_stock_counted hay backflush.
Vay nen cac nguyen lieu PREP/JELLY se duoc import nhu nguyen lieu truc tiep de POS co the tru kho don gian.

Vi du:
- Cot cafe phin duoc xem nhu mot ingredient rieng.
- Nuoc duong duoc xem nhu mot ingredient rieng.
- Thach cafe duoc xem nhu mot ingredient rieng.

Day la cach lam co y de giu database don gian, de test, de giai thich va phu hop voi mon SWT.

File import cong thuc lam gi?
-----------------------------
File supabase_recipe_seed_from_old_project_v1.sql chi import data.
No lam cac viec sau:

1. Tao TEMP TABLE de chua du lieu cong thuc tu file cu.
2. Insert / update ingredients.
3. Insert / update products.
4. Insert / update recipes.
5. Insert / update recipe_items.
6. Gop duplicate ingredient trong cung mot recipe bang SUM(quantity_required).

File nay KHONG lam cac viec sau:
- Khong CREATE FUNCTION sp_add_recipe.
- Khong CREATE TRIGGER tru kho.
- Khong tao function backflush.
- Khong tao function xu ly PREP/JELLY ve nguyen lieu cha.
- Khong tao function tinh order.
- Khong tu dong tru kho khi insert order.

Trigger/function hien co trong schema
-------------------------------------
Database schema v1 chi co function phu tro:

set_updated_at()

Va trigger updated_at cho mot so bang:
- products
- ingredients
- recipes
- orders
- app_users

Cac trigger nay chi dung de cap nhat cot updated_at khi sua record.
No khong lien quan den cong thuc, khong tinh tien, khong tru kho.

Logic order nam o dau?
----------------------
Toan bo nghiep vu order phai nam trong backend Node.js/Express.
Backend phai xu ly theo flow:

1. Staff submit cart.
2. Backend check user da login.
3. Backend check role la STAFF.
4. Backend check cart khong rong.
5. Backend lay danh sach product.
6. Product phai active va chua bi deleted.
7. Product phai co recipe active.
8. Backend lay recipe_items.
9. Backend tinh total amount.
10. Backend gom tong nguyen lieu can tru.
11. Backend check ingredients.current_stock co du khong.
12. Neu thieu stock: tra loi loi, khong tao order, khong tru kho.
13. Neu du stock: tao order.
14. Tao order_items.
15. Tru ingredients.current_stock.
16. Tao stock_transactions type ORDER_DEDUCT.
17. Commit transaction.
18. Tra ve order_code va total_amount.

Cach chay SQL import cong thuc
------------------------------
Cach nhanh nhat:

1. Mo Supabase Dashboard.
2. Vao SQL Editor.
3. New query.
4. Paste noi dung file supabase_recipe_seed_from_old_project_v1.sql.
5. Run.

Cach chay bang backend:

1. Copy recipeSeed.js vao backend/src/seed/recipeSeed.js.
2. Dam bao backend/.env co DATABASE_URL.
3. Them script vao backend/package.json:

   "seed:recipes": "node src/seed/recipeSeed.js"

4. Chay lenh:

   npm run seed:recipes

Cau lenh check sau khi import
-----------------------------
Chay trong Supabase SQL Editor:

select count(*) as ingredients_count from ingredients;
select count(*) as products_count from products;
select count(*) as recipes_count from recipes;
select count(*) as recipe_items_count from recipe_items;

Check product theo size:

select name, price, status
from products
where name ilike '%Phin Den Da%'
order by name;

Check cong thuc mot mon:

select
  p.name as product_name,
  i.name as ingredient_name,
  ri.quantity_required,
  i.unit
from products p
join recipes r on r.product_id = p.id and r.deleted_at is null
join recipe_items ri on ri.recipe_id = r.id
join ingredients i on i.id = ri.ingredient_id
where p.name = 'Phin Sua Da - M'
order by i.name;
